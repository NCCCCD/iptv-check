# AGENTS.md｜IPTV播放列表检测

## 1. 项目信息
- 项目名称：IPTV播放列表检测
- 项目代号：iptv-check
- 项目类型: cli
- 当前项目状态：已立项，待启动
- 当前轮次：第一轮
- 当前优先级：中

## 2. 项目目标
- 核心目标：检测 IPTV M3U 播放列表中每条流的可达性和播放效果
- 目标用户：本人（维护自用 IPTV 列表）
- 主要场景：将 GitHub 上的 M3U 播放列表拉下来，逐条检测流是否可用
- 成功标准：输入 GitHub 共享链接即可输出一份完整的可达性报告

## 3. 本轮范围
### 必做（In Scope）
- M3U 文件解析（频道名、URL、分组、Logo）
- DNS 解析 + TCP 连接可达性
- HTTP 响应状态码 + 响应时间
- 支持 GitHub 共享链接自动转 raw
- 并发检测（可配置）
- Markdown / JSON 两种报告格式

### 暂不做（Out of Scope）
- 图形界面
- 自动定时检测
- 流协议深测（RTMP、UDP 等）

## 6. 关键上下文（反脆弱沉淀）
- rtp2httpd 会重写 M3U：`rtsp://...` → `http://10.10.2.6:5140/分组/频道/catchup?...`
- iStoreOS/OpenWrt 重启**会清除静态路由**，必须在 `/etc/rc.local` 持久化
- 路由丢了的症状：所有频道回放挂（rtp2httpd 返回 503），但直播正常
- 修复命令：`route add -net 182.139.0.0 netmask 255.255.0.0 gw 10.187.224.1`
- APTV 占位符格式：`${(b)yyyyMMddHHmmss}`（北京时间），Alexander Sofronov IPTV 不支持 RTSP catchup
- 全频道无回放查路由，部分频道无回放查 merge 日志
