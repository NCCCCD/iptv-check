# PROJECT.md｜IPTV播放列表检测

## 使用约定
1. 项目名：IPTV播放列表检测
2. 项目代号：iptv-check
3. 一句话目标：输入 GitHub 共享链接，输出 M3U 播放列表可达性报告
4. 这是当前项目唯一主文档；项目对话里发送 `1` 时优先读取这里

## 当前共识
1. 背景：本人维护 IPTV 播放列表（GitHub 上），列表内有很多流，无法逐条手动检查是否可用
2. 核心目标：做一个 CLI 工具，输入 M3U 链接即可批量检测
3. 运行平台：Mac 本地运行
4. 已出第一版草稿脚本，待项目线程中打磨

## 范围与边界
### 本轮必做
1. M3U 解析 + 多种协议 URL 支持
2. 并发流检测（DNS / TCP / HTTP）
3. GitHub 共享链接自动转 raw
4. 报告输出（Markdown / JSON）
5. 可配置并发数、超时时间

### 暂不做
1. GUI
2. 定时监控
3. 非 HTTP 协议深测

## 当前风险
1. 部分 IPTV 流使用非标准端口或协议，可能误判
2. 防盗链 / User-Agent 检测导致 HTTP HEAD 不准确
3. 大型播放列表（1000+ 条）并发检测可能触发流服务器反爬

## 当前有效决策
1. 使用 Python 3.8+，零第三方依赖
2. 使用 ThreadPoolExecutor 实现并发
3. 四档状态：✅ ⚠️ 🟡 ❌
4. GitHub 链接用 `--github` 参数显式声明

## 执行方式
1. 进入项目对话后，可以直接发送 `1`，系统先读取本文件
2. 提修改意见后，系统默认更新项目根目录下的 `修改意见.md`
3. 处理修改意见前，先查看项目根目录下最新截图（不适用 CLI 工具则跳过）
4. 若本轮只是同步修改意见，窗口默认只回：`已同步更新修改意见。`
